# AI Agent Instructions

You are helping build this project end-to-end.

Always follow these rules:

1. Read README.md and requirements.md before coding.
2. Do not make random assumptions. Ask questions if requirements are unclear.
3. Build feature by feature.
4. Use clean folder structure.
5. Keep code simple enough to explain in an interview.
6. After backend changes, provide how to run and test APIs.
7. After frontend changes, provide how to run and test UI.
8. Do not hardcode secrets.
9. Use meaningful Git commit messages.
10. Prefer working software over over-engineering.

## Tech Stack

Frontend:
- React
- Plain CSS or Tailwind if already configured

Backend:
- Java Spring Boot
- Spring Web
- Spring Data JPA
- PostgreSQL Driver
- Lombok optional

Database:
- PostgreSQL

## Workflow

1. Understand requirements.
2. Create backend.
3. Create patient entity.
4. Create repository.
5. Create service.
6. Create controller.
7. Test APIs.
8. Create frontend UI.
9. Connect frontend to backend.
10. Run end-to-end test.
11. Commit changes.
